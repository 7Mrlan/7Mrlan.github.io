class

发表于 2022-08-06 更新于 2022-08-07 分类于 JavaScript

引言

class是ES6引入的新概念，它作为对象的模板，生成需要的实例化对象，比起传统的构造函数，它更加的容易理解，对于原型链和继承关系的描述也更加直观

(需要说明的是，class的特性不是新的继承方式，引擎的底层依然是原型，它只是提供了一种更直观，更容易创键原型链的方式，而且十分接近传统的OPP{面对对象编程})

class详细介绍

上面一堆文字可以不用看，我会抛出一些有利于理解的概念帮助掌握class

  1. 问 class是干什么的？

答： 用来作为实例化对象的一个模板，和构造函数的功能相同

  1. 它是新的继承方式或者功能吗

答：并不是，它的功能和构造函数一样，继承方式的底层依然是原型链

从第二个问题和我举的例子应该很容易看出来，class就是程序员小哥哥帮你把超级复杂的原型和原型链这种很难的东西封装成传统的类对象，增加了代码的可读性，也帮助你在复杂的原型链绕的找不着北

现在只需要

很清晰的就能理清sonClass是fatherClass的子类，它继承于fatherClass，那么在它里面找不到的属性和方法就会沿着这条继承链去它爸爸的属性和方法里面找，还找不到就去爷爷类里面找依次往上，要么找到结束，要么找到这条继承链的上游终点（maybe 是你太太太太爷爷）

我们现在可以说只是明白了类的作用，但其实已经足够了，它就是很单纯的作为一个实例化对象的模板，我重复了很多次这句话。那么下面我就会完整的去重头开始介绍它的使用方法和注意事项。

class的书写规范

class后跟类名加一对大括号,里面有个构造函数如果你不写，默认的构造函数会被生成。

先来看构造函数

  1. 它的作用

答：初始化变量。

  1. 它干了什么
  1. 创建了一个空对象
  2. 将this指向了它创建的这个对象
  3. 执行构造函数中的代码，初始化变量
  4. 返回这个对象

当我们new一个实例的时候，所接收的返回值就是constructor为我们创建的对象

再看看类方法

在以往用function来创建构造函数，我们一般只在它的内部定义属性，方法定义在它的prototype上。在类里面定义方法几乎等于这个过程，我们可以举个例子，比如打印学生名字

函数里面的this指向也是这个对象，构造函数干的几件事我上面已经叙述完了，在类中还看见了我在constructor上面声明了name属性，这里的声明是可选的，因为类最先执行的是constructor，这样声明只是为了可读性，更容易确定哪些属性属于类本身，因为我们知道当一个类中属性被我们访问，如果本类中没有，就会上溯原型链（在class中叙述为继承链更容易理解）去它的父类里面找，依次往上。所以你在实例中访问到的一些属性和方法并不一定是你类本身就拥有的。

讲讲继承

上面已经叙述了class类的书写规范，之后也会涉及私有变量或私有方法的定义方法，还有静态变量的创建，但是不要着急好兄弟，在这之前我们先来聊一聊继承

继承的关键字是extends，用它来声明一个类继承自另一个类。我们看见这个单词加了s，这是因为一个类可以是多个类的子类，当然，一个类也可以是多个类的父类

举个例子（我用水果来举例子，毕竟个人水果重度爱好者）

定义一个水果类，它有种类和产地两项属性，另外加上一个获得水果产地的方法，这个时候如果我们想创建子类水果，因为同一品种也有很多种可以使用extends去继承它的父类水果，因为水果本身都有一些共同的特性，我们不想每次都去定义这些，直接继承是最好的方法

我们在子类的构造函数里，先调用父类的构造方法，species和orrgin会先在父类中被设置。然后构造函数的其他属性都会被设置，子类中可以覆盖父类的方法，只要取相同的函数名即可，因为当本来中有某种属性和方法不会在上溯原型链，去父类中寻找了

简单来说继承就是父类中定义的属性和方法你可以访问，因为比如我还想要香蕉，哈密瓜，它们都有父类共同的那两个属性，重复定义不仅浪费空间，还会让代码看起来一团糟

私有关键字#

js中用#代表私有变量或者私有方法，只有在变量或方法名的最前面加#即可，私有属性不能再类外面访问，只能通过类里面的方法互相访问，和闭包有异曲同工之妙，它负责隐藏变量和方法,它往往用来定义不愿意让别人修改的变量
