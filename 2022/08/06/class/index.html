class

发表于 2022-08-06 分类于 JavaScript

引言

class是ES6引入的新概念，它作为对象的模板生成需要的实例化对象，比起传统的构造函数，它更加的容易理解，尤其的对于原型链和继承关系也更加直观

(需要说明的是，class的特性不是新的继承方式，引擎的底层依然是原型，它只是提供了一种更直观，更容易创键原型链的方式，而且十分接近传统的OPP{面对对象编程})

class详细介绍

上面一堆文字可以不用看，我会抛出一些有利于理解的概念帮助掌握class

  1. 问 class是干什么的？

答： 用来作为实例化对象的一个模板，和构造函数的功能相同

  1. 它是新的继承方式或者功能吗

答：并不是，它的功能和构造函数一样，继承方式的底层依然是原型链

从第二个问题和我举的例子应该很容易看出来，class就是程序员小哥哥帮你把超级复杂的原型和原型链这种很难的东西封装成传统的类对象，增加了代码的可读性，也帮助你在复杂的原型链绕的找不着北

很清晰的就能理清sonClass是fatherClass的子类，它继承于fatherClass，那么在它里面找不到的属性和方法就会沿着这条继承链去它爸爸的属性和方法里面找，还找不到就去爷爷类里面找依次往上，要么找到结束，要么找到这条继承链的上游终点（maybe 是你太太太太爷爷）
