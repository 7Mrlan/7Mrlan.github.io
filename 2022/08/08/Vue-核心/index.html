Vue-核心

发表于 2022-08-08 更新于 2022-08-14 分类于 Vue

官方解释

Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动
source:https://cn.vuejs.org/v2/guide/

以上是官方对vue的描述，这里提到的渐进式框架，我举个例子帮助理解什么是渐进式，如果你的公司以往都是使用Jquery框架，现在想更换vue框架，你可以逐步的删除关于Jquery的代码，然后逐步用Vue进行项目重构。当然这并没有那么重要。我觉得比较重要的有三个方面

  • 响应式
  • 声明式编程
  • MVVM框架

我会逐步对他们进行展开讨论的，你只需知道现在它们会是我写博客的重点部分，我们暂时不讨论它们

上面这些可以自行去官网上查找，我们直接进入正题

在我们引入的vue.js中，已经为我们定义好了Vue类，对类不明白的可以去看我的class类的介绍，我们有了Vue这个类作为模板，去创建实例化的对象，我们将对象作为参数传到构造器里。这里我们传了两个属性

  1. el代表元素element的缩写，它的作用是将vue实例挂载到DOM对象上，一旦挂载了vue实例，这里DOM就能被vue进行管理解析，vue即会进行数据绑定，当数据发生变化的时候它会响应式的更新DOM，同时也会对DOM进行监听，当DOM的事件的触发使数据发生改变时，vue也会实时的去更新data里面的数据，这就是MVVM模型
  2. data代表数据，它可以由原生js传入静态对象作为数据，也可以传入向服务器请求的数据它属于MVVM模型中的model

利用这两个属性，vue就可以对它们同时进行管理，使得数据能响应式的更新，你可以试着在控制台输入。app.message = ‘鸡你太美’，我们这个时候可以发现DOM的内容同步的更新了。

但是对于声明式编程，我们只需要考虑结果，我们需要什么？上面的例子需要message，我们只需要用插值表达式插入即可，其他的事情都交给Vue来帮我们管理解析。
这里值得注意的是，我们知道网页解析的时候，当script还没有加载的时候，网页是由这样显示的，当加载到el：‘#app时，Vue对id为app的DOM进行管理解析，会自动的管理数据和DOM匹配等待操作，声明式的好处就是可以做到数据页面相分离，我们不用自己操作页面的textContent去更改页面了，我们可以专注于数据和页面分开进行操作

哪些数组的方法是响应式的

变更方法
Vue 将被侦听的数组的变更方法进行了包裹，所以它们也将会触发视图更新。这些被包裹过的方法包括：

  • push()

  • pop()

  • shift()

  • unshift()

  • splice()

  • sort()

  • reverse()
    你可以打开控制台，然后对前面例子的 items 数组尝试调用变更方法。
    因为这些方法修改的都是data里面数组本身的数据，不是生成一个新的数组的，就比如字符串方法的slice或者toLoweCase,toUpperCase,这些都不修改原字符串，这就属于不修改本身数据，字符串也有一些方法只是生成新数组的，如filter()、concat() 和 slice()。它们不会变更原始数组，而总是返回一个新数组。当使用非变更方法时，可以用新数组替换旧数组

v-on

之前提到前缀v-的就属于指令，指令会被vue实例所接收处理。上面的v-for是循环操作数据的，v-on的功能是添加监听的。我们之前就说过vue是MVVM框架，不仅负责数据的绑定响应式的更新DOM，而且可以监听DOM去对数据进行响应式更新，添加监听是对DOM元素的操作，属于这类范畴

例：计数器，当点击加号按钮数据加一，点击减号数据减一

vue处理v指令，解析，v-on用来监听DOM事件，当指定的事件发生时，执行方法（方法通常是methods中定义的方法）

Vue的生命周期

生命周期：事物从诞生到消亡的整个过程

Vue生命周期:Vue的诞生源自你创建一个vue实例

以上是尤雨溪，Vue源码中的一小段。我们可以清楚的看见这个构造函数的冰山一角。但它现在只是构造函数的状态，也仅仅是一个模板。并没有诞生。

当我们创建Vue实例的时候，它的生命开始了。人的诞生到结束会做很多的事情，比如六岁上小学，12岁上初中，15岁上高中，18岁上大学，后面漫长的人生都会做一系列的事情。Vue和人一样，它在自己的生命周期也会完成很多的事情。我们看Vue源码就可以看见极其多的初始化和一系列数不清的代码，这些都是它将要完成的事情。我们希望它能在它特定的生命周期告诉我们它到哪一步了，以便于我们做某些操作。这很重要。举个例子，如果在el元素还没有挂载到Vue实例里面的时候，我们对它做的任何事情都没有意义，比如你让一个刚出生的婴儿结婚生子，这绝对的荒唐。所以，Vue会通过一些特定的属性告诉你它的生命周期执行到哪一步了。你只需要在你传入的options里面的特定属性上写入在它某个生命周期你希望它完成的事情，到了那个特定的周期，它会来解析你的option，找到那个特定生命周期你期望它完成的事情去完成。这些特定的属性叫做生命周期函数

生命周期函数

callHook,钩子函数，在Vue特定的生命周期用这个函数可以回调options里面的函数

我们看这张图示可以看见，当Vue在它的生命周期的特定阶段，红色的框代表的就是生命钩子，它利用callHook生命钩子回调我们options传入的特定函数，比如created,mounted,updated函数等等，这些我们都可以在optons里面设置具体的回调函数所要执行的内容
